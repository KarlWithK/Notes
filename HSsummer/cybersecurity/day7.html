Operating Systems Concepts

A computer model

A operating system has to deal with the fact that a computer has different parts such as a CPU, RAM, IO, and long-term storage.

OS Concepts

An operating system (OS) provides interface between the users of a computer and that computer's hardware:

  • Manages the ways applications access the resources
  • Manages multiple people and programs.

Multitasking

Give each program a "slice" of the CPU's time

  • The cpu is running so fast that to any user it appears that the computer is running all the programs simultaneously.

The Kernel

The Kernel is the core component of the OS. It handles the management of low-level hardware resources like memory and IO.

  • Most OS define the tasks associated with the kernel in terms of a layer metaphor, with the hardware component being at he bottom and user applications at the top.

Input / Output

The input/output devices of a computer include things like keyboard and network card etc.

  • Each such device is represented in an operating system using a device driver, which encapsulates the details of ow interaction with that device should be done.
  • The Application programmer interface (API), which the device drivers present to applications programs, allows those programs to interact with those deice at a fairly high level, while the OS does the work.

System Calls

User applications interact with with the kernel using a system called system calls. A system call is a way for programs to interact with the operating system.

  • Systems calls are usually contained in a collection of programs, that is, a library such as the c library (libc).
  • System calls include those for performing file I/O

Processes

A process is an instance of a program that is currently executing

  • The actual contents of all programs are initially stored in persisted storage, such as a hard drive.
  • In order to be executed, a program must be loaded into RAM and uniquely identified as a process.
  • In this way, multiple copies of the same program can be run as different process.

Process IDs

Each process running on a given computer is identified by a unique nonnegative integer, called the process ID (PID)

  • Given the PID for a process, we can then associate its CPI time, memory usage, user ID (UID), and program name

File Systems

A file system is an abstraction of how the external, nonvolatile memory of he computer is organized.

  • Operating systems organize files hierarchically into directories which can contain more.
  • Thus, a volume, or drive, consist of a collection of nested directories that form a tree.
  • The topmost directory is called the root directory. (In windows c:/)

File Permissions

File permissions are checked by the OS to determine if a file is readable, writable, or executable by a user or group of users.

  • In Unix based OS's, a file permission matrix shows who is allowed to do what -> ls -l:
    • Files have owner permissions, which show what the owner can do, and group permissions, which show what some group ID can do, and world permissions, which give default access rights.

Memory Management

The Ram memory of a computer is its address space.

  • It contains the cod for a program, its input data, and its working memory.
  • For any running process it keeps its parts in separate address space.
  • Security requires we never mix up these different segments

Memory Organization

Each segment should be separated

  • Text: contains the actual (binary) machine code of the program
  • Data: Contains static program variables that have been initialized in hot program code.
  • BSS: Named for an antiquated acronym for block started by symbol, contains static variables that are uninitialized.
  • Heap: Known as the dynamic segment stores data generated during the execution of a process.
  • Stack: Houses a stack data structure that grows downwards and is used for keeping track of the call structure of subroutines like methods and their arguments.

Virtual Memory

There is generally not enough computer memory for the address space for all running processes, so the OS sometimes uses the hard drive as virtual memory. This gives each process the illusion that it has acess to its complete address space.

  • The OS supports this view, but it is not how it is organized.
  • Memory is divided into pages, and the OS keeps track fo which ones are in memory and which are in disk.:
    • There can be page faults.

Virtual Machine

A view that an OS presents that a process is running on a specific architecture and OS, when its not. Benefits:

  • hardware efficiency
  • portability
  • security
  • management
