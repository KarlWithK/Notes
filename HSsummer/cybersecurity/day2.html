The Ten Security Principles

Economy of mechanism

This principle stresses simplicity in the design and implementation of security measures

  • Everyone should be able to use it.
  • Even kids(?)

Fail-safe defaults

The principles states that the default configuration of a system should have a conservative protection scheme.

  • A new user should have default that give it minimum about of rights
  • Web browsers have way too many right

Complete Mediation

The idea behind this principle is that every access to a resource must be checked with compliance with a protection scheme.

  • Be wary of techniques that save your password

Open design

The security architecture and design of a system should be made publicly available

  • Security should rely only on keeping cryptographic keys secret
  • open design allows for a system to be scrutinized by multiple parties, which leads to the early discovery and correction of security vulnerabilities caused by design errors.
  • Opposite of security by obscurity, which tries to achieve security by keeping everyone secret.

Separation of privilege

Dictates that multiple conditions should be required to achieve access to restricted resources or have a program perform some actions.

Least Privilege

Each program and user of a computer system should operate with the bare minimum privilege necessary to function properly.

  • Abuse of privileges is restricted and any damage is minimized.
  • Need-to-know is a similar technique in the military.

Least common Mechanism

Systems with multiple users, mechanism allowing resources to be shared by more than one user should be minimized.

Psychological acceptability

Principle states that user interfaces should be well designed and intuitive, and all security-related settings should adhere to what an ordinary user might expect.

Work factor

According to this principle, the cost of circumventing a security mechanism should be compared with the resources of an attackers when designing a security scheme.

  • Student grades don't need that much security compared to military

Compromise recording

Sometimes it is more desirable to record the details of an intrusion than to adopt more sophisticated measures to prevent it.

  • Camera are better than traps sometimes to take note of the incident
  • Log files are nice

Cryptographic concepts

Encryption

Establish confidential communication over an insecure channel that is subject to eavesdropping.

  • A message is called plain text and using an encryption algorithm it outdoes a cypher text.
  • This allows messages to be transmitted in insecure channels.

Cryptosystems

  1. The set of possible plaintexts
  2. The set of possible ciphertexts
  3. The set of encryption keys
  4. The set of decryption keys
  5. The correspondence between encryption keys and decryption keys
  6. The encryption algorithm to use
  7. The decryption algorithm to use

Casar Cipher

An algorithm that replaces each letter with one "three over in the alphabet"

Symmetric Cryptosystems

Use the same key to encrypt and decrypt.

Requires each pair of communicating parties with same key.

  • Formula: n(n-1)/2

Public-Key Cryptography

Has a public and secret key. Encryption is done by public key and decryption is done by secret key.

Digital signatures

Public-key encryption provides a method for doing digital signatures.

  • To "sign" a message you encrypt it with your public key

Cryptographic Has Functions

Message Authentication Codes

Digital Certificates

Certificate Authority (CA) digitally signs a binding between an identity and the public key for that identity.

Social Engineering

  • Pretexting: Creating a story that convinces an administrator or operator into revealing secret information.
  • Baiting: offering a kind of "gift" to get a user or agent to perform an insecure actions
  • Quid pro quo: offering an action or service and then expecting something in return.
