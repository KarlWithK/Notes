Firewalls, Tunnels, and Network Intrustion

Firewalls

A firewall is an integrated collection of security measure destined to prevent unauthorized electronic access.

Firewalls Polices

To protect private networks and individual machines from the dangers of the great internet, a firewall can be employed to filter incoming or outgoing traffic based on a predefined set of rules called firewall policies.

Policy Actions

Packets flowing through a firewall can have one of three outcomes:

  • Accepted
  • Dropped: Not allowed and no indication of failure
  • Rejected

Polices used by the firewall to handle packets are based on several properties of the packets being inspected, including the protocol used such as:

  • TCP or UDP
  • The source and destination IP address
  • More

Blacklists and white lists

There are two fundamental approaches to creating firewall polices (or rule sets) to effectively minimize vulnerability to the outside world while maintaining the desired functionality for the machines in the trusted internal network

  • Blacklist approach or white list

Firewall Types

Packet filters (stateless): If a packet matches the packet filter's set of rules the packet filter will drop or accept it

  • Stateful filters: can determined if a packet is either the start of a new connection, a part of an existing connection, or is an invalid packet.
  • Application layer: works like a proxy

Stateless Restriction

May have to be fairly restricted in order to prevent most attacks

State full Firewalls

Can tell when packets are part of legitimate sessions originating within a trusted network

  • Only requested TCP connections

Tunnels

The contents of TCP packets are not normally encrypted, so if someone is eavesdropping on TCP connection, he can often see the complete contents of the payloads in this session.

  • One way to prevent this is to use tunneling protocol: where the communication between a client and server is automatically encrypted, so that useful eavesdropping stews.

Secure Shell

CLI to connect to the server using a TCP session with secret-key exchange to establish.

IPSec

Defines a set of protocols to provide confidentiality and authenticity for IP packets

  • Each protocol can operate in one of two modes, transport mode or tunnel mode
    • In transport mode, additional Apses header information is inserted before the data

Virtual Private Networking (VPN)

VPN is technology that allows private networks to be safely extend over long physical distances by making use of public network, such as the internet, as a means of transport

TYPES of VPNs

Remote access VPNs allow authorized clients to access a private network that is refereed to as an intranet.

  • Site-to-site VPN solutions are designed to provide a secure bridge between two or more physically distant networks.

Instrusion Detection Systems

Intrusion: aims at compromising the security of the system.

IDS Components

The IDS manager compiles data from the IDS sensors to determine if an intrusion has occurred.

  • This determination is based on a set of site polices, which are rules and conditions.

Types of Intrusion Detection Systems

  • Rule-Based Intrusion Detection: identify the types of actions that match certain known profiles and would encode a signature for such an attack.
  • Statical Intrusion Detection:
    • A profile is built, which is statistical representation of the typical ways that a user acts or host is use. It uses this profile to determine threshold for anomalous behaviors and then sound an alarm any time a user or host deviates significantly from the stored profile for that person or machine.
