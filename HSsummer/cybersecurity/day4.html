Computer Forensics

Scientific process of persevering, identifying, extracting, documenting, and interpreting data on a computer.

  • Used to obtain potential legal evidence.

The forensic Paradigm

4 Stages: Identification, Collection, Analysis & Evaluation, and Reporting

Identification

  • Identification: identity specific objects that store important data for the case analysis:
    • You only care about the data -> hard drive

Collection

Collection establish a chain of custody and document all steps to prove they the data collected data remains intact and unaltered:

  • maintaining the chain of custody:
    • Maintain possession of all objects
    • must be able to trace evidence back to source
    • prove source integrity
  • Priority by volatility:
    • Some data is more volatile
    • RAM > swap > disk > CDs/DVDs
    • capture more volatile evidence first.
  • Using only tools and methods that have been tested and evaluated to validate their accuracy and reliability.

Why use images

  • Information on digital media is easily changed.
  • Once changed it is usually impossible to detect that a change has taken place unless other measures have been taken
  • Calculate cryptographic has to establish a check point
  • examining a live file changes state of the evidence
  • The media is the crime scene.
  • Protecting the crime scene is paramount as once evidence is contaminated, it cannot be decontaminated -> only one try
  • A digital image copy allows preservation of the original evidence, prevents alteration of original evidence and allows recreation of the data.
  • A file copy does not do the above

Analysis and Evaluation

Determination of the type of information stored on digital evidence and conduct a through analysis of the media

  • Know where evidence can be found
  • Understand techniques used to hide or "destroy" digital data
  • toolbox of techniques to discover hidden data and recover "destroyed" data
  • cope with HUGE qualities of digital data
  • Ignore the irrelevant, target the relevant
  • Thoroughly understand circumstances which may make "evidence" unreliable:
  • Where is the evidence:
    • Undeleted files, expect some names to be incorrect
    • Deleted files
    • windows registry
    • print spool files
    • hibernation files
    • temp files
    • slack space:
      • Slack space is the space between the logical end of the file and the physical end of the file.
    • Internet history.

Reporting

Prepare and deliver an official report

  • Describe the incident
  • Be able to withstand decision makers
  • offer valid conclusions, and opinions, or recommendations.

Digital forensics tools

Command line tools that are guaranteed not to alter the disk:

  • HELIX: a live cd with plenty of forensic tools ready to be used
  • ENCASE: A series of proprietary forensic software produced by Guidance software

How to hide data

  • Cryptography
  • Stenography: the process of hiding data inside other data
  • Change file names and extensions. .doc -> .tmp
  • Hidden tracks:
    • Most hard disks have # of tracks hidden
    • They can be used to hide/read data by using a hex editor
  • Deleted files

Bitstream vs Backups

  • Forensics copies (Bitstream):
    • Bit for bit copying captures all the data on the copied data
    • Including hidden and residual data
    • Often the "smoking gun" is found in the residual data

Anti-Forensic and data security

There are techniques that try to prevent forensic investigators to do their work.

  • Securely deleting data, so that it cannot be restored with forensic methods
  • Prevent the creation of certain data in for first place
  • Data which was never there, obviously cannot be restored with these methods.
  • A desgausser Magnetic Field or Shredder or termite can destroy the media.
  • Just wipe the media with a simple erase or destructive erase
