SYN Flood

  • Typically DOS attack, through can be combined with other attack such as TCP hijacking
  • Rely on sending TCP conneciton requests faster than the serer can process them
  • Attacker creates a large number of packets with spoofed source addresses and setting the SYN flag on these
  • The server resonds with a SYN/ACK for which is never gets a response (waits for about 3 minutes each)
  • Eventually the sever stops accepting connection requests, thus triggering a denial of service
  • Can be solved in mutiple ways
  • One the common way to do this is to use SYN cookies.

TCP Data Transfer

During connection initalization using the three way handshake, inital sequence numbers are exchanged

  • The TCP header includes a 16 bit checksum of the data and parts of the header, including the source and destination.
  • Acknowledgment is used by TCP to keep track of network congestion and control flow
  • TCP connections are terminated with a 4-way handsahke:
    • The client which wishes to terminate the conection sends a DIN message to the other client
    • The other client responds by sending an ACK
    • the other clients sends a FIN
    • the original client now sends an ACK, and the connections is terminated.

TCP congestion Control

  • During the 80s, uncontrolled TCP messages casued large scale network congestion

Optimistic ACK attack

  • Takes advantage of the TCP congestion control
  • Sends flood of ACKS and the server used bandwidth that it does have
  • Finally the network is no longer reachable

IP Spoofing

An attempt by an intruder to send packets from one IP address that appear to originate at another. Blind spoofing:

  • Attack from any source

Non blind spoofing:

  • Attacker is from same subnet

Blind IP Spoofing

The TCP/IP protocol requires that "acknowledgment" number be sent across sessions

  • Makes sure that the client is getting the server's packets and vice verser

NON-blind

IP spoofing without inherit wetly knowing the acknowledgments sequence pattern

  • Done on the same subnet
  • Use a packet sniffer to analyze the sequence

User datagram Protocol

UDP is stateless, unreliable datagram protocol built on top of IP, that it lies on level 4

  • It does not provide delivery guarantees, or acknowledgments, but it is faster
  • Can distinguished data for multiple concurrent applications on a single host.

Network address translation

Alleviate IPv4 address space congestion

  • Relies on translating address in an internal network, to an external address that is used for communication to and from the outside world
  • NAT is usually implemented by placing a router in between the internal private network and the public network
