[ bash/zsh ] - this might not work on dash or fish.

  • Ctrl-R -> search through your history
  • !! to reuse the previous command for the current command.
  • cd - to go back to the previous directory.

auto cd - I often find myself moving through a lot of directories. And it's really a a pain to write cd all the time (yes i'm lazy). To get rid of this. Use AUTOCD

  • bash - set in your bashrc shopt -s autocd
  • zsh - setopt AUTOCD

vi mode - by default bash/zsh come with emacs like binding. So if you use vim you might be interested in vi mode.

  • bash - set -o vi
  • zsh - bindkey -v

Now if you do use vi mode then you can use <Esc>v to write your command(s) in a vim buffer. Quit and the command should run.

[ ctags ] If you are writing c or any language then I recommend using tags. I personally use universal ctags. Tags allow my to go through my code very easily. Its like the "Go to definiton" many IDEs have.

  • https://en.wikipedia.org/wiki/Ctags

[ git ] gitignore - Everyone here should know about .gitignore - you are kinda required to use it for any project. But did you know that you can have a global gitignore? Just edit your .gitconfig and under [core] add excludesfile = $SomeDir. For example I have it in my .config/git as so excludesfile = /home/karl/.config/git/gitignore_global. A global gitignore is really useful for things you never want to upload to your master / version conrol host. For example, I have git always ignore .class for java, tag files and for javascript folks I have node_modules/ to never, never, never have to worry about uploading 1000 MBs of files or more.

tools - You can make custom tools for git. For example I currently use vim's diff to see any changes in my current git repo: 

[diff]
	tool = nvimdiff
[difftool "nvimdiff"]
	cmd = nvim -d $LOCAL $REMOTE
  • Now everytime I run git diff it uses vim to show the changes. You can use use this for merge tools

git hooks - Say you want a script to run a change happens in your repo. You can use git hooks to do that. I currently use it to regenerate tags for my repo when I new file is added.
