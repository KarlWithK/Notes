Comparing sorts (6.4)

We have looked at two sorts that use different algorithms to accomplish at the same task. However which one is the better algorithms? There are several reasons to prove efficiency:

  • Simple algorithms vs. Complex algorithm,
  • Speed, especially dealing with many items,
  • Memory, less the better,
  • Works well with the data at hand.

CPU -> ALU - Registers (Hold Bits)

-> Level one cache -> L2 cache -> RAM -> Main memory

Note; There are more efficient algorithms than the selection, and insertion sort, but they're more complex and harder to understand

Time Efficiency refers to how long it takes an algorithm to run.

Space efficiency referees to the amount of space an algorithm uses.

An algorithm that uses less space and runs faster is more efficient.

Time efficiency of an algorithm is calculated by counting the number of statements executed during a run. As the input (N) grows, we don't need to count individual statements but instead we look how the number of comparisons grows.

Listing

For example, selection sort has two loops, the outer loop will run (N - 1) times, and the inner loop runs 2 to (N - 1) times

n(n -1 ) / 2 = # of comparisons (n^2 - n) / 2 = # of comparisons

Now notice as N becomes very large, n2 grows much faster than n, so fast that we can ignore n, and just say the algorithm is on the order of n2.

Analyzing insertion sort, for an input size of n, ow many times does the outer loop run? How many times will the inner loop run for the bes case? Worst case?

Outer loop will run (n - 1) times.

Insertion Sort is O(n2) The running time is no worse than on the order of n2 units of time.

Space efficiency we look at the amount of memory needed. When sorting, we are just moving the elements around, therefore we just need n memory location, or the size of the array (we don't consider the space for temporary variables, because the number is constant) APCS
